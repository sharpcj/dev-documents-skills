## 模版说明

### 文档定义

| 文档类型 | 用途 |
| --- | --- |
| **技术方案** | 用于单个特性版本的方案撰写和评审 |
| **概要设计** | 类似app的整体说明书，在固定的周期做更新，将评审通过的技术方案通过链接或重写的方式合入概要设计中 |

### 必填章节（不可删除）

| 章节 | 说明 |
| --- | --- |
| **1. 背景** | 包含项目背景和应用背景两个子章节 |
| **1.1 项目背景** | 说明项目优先级（首发/必须上线/建议上线） |
| **1.2 应用背景** | 说明基于哪个版本开发 |
| **2. 需求描述** | 提供PRD、ERGO、GD的需求链接 |
| **3.1 方案描述** | 详细描述技术方案的具体实现方式 |
| **3.2 业务流程图** | 用户视角的业务流程，标注需求完成前后对用户的业务影响 |
| **3.3 数据流程图** | 数据方面的流转方案 |
| **3.4 技术架构拓扑图** | 应用内部技术模块之间和内外部模块之间的技术架构 |
| **3.5 关联模块具体方案描述** | 描述与framework、驱动、其他应用等模块的技术架构和接口方案 |
| **3.6 异常和边界场景** | 以表格形式列出异常情况、结果、描述和验证次数 |
| **3.7 方案劣势、风险和解决措施** | 列出方案的劣势、潜在风险及对应的解决措施 |

### 使用规范

- 斜体内容为**示例**，编写时替换为实际内容
- 已列出的一、二、三级标题**必须包含，不可删除**
- 可根据项目需要**新增**其余模块
- 使用前需**复制到自己项目空间**后再编写
