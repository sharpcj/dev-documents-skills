## 模版说明

### 必填列（不可删除）

需求矩阵表格包含以下 **12 列**，均为必填项，不可删除：

1. **一级功能** - 功能模块的一级分类。
2. **二级功能** - 功能模块的二级分类。
3. **三级功能** - 功能模块的三级分类。
4. **需求功能描述** - 对该功能的详细描述。
5. **前置依赖条件** - 完成该功能所需的前置条件及依赖。
6. **开发工作量（人天）** - 预估的开发人天。
7. **开始时间** - 开发开始日期。
8. **完成时间** - 开发完成日期。
9. **负责人** - 该功能的开发负责人。
10. **适用端/平台** - 功能适用的端、平台或系统类型，如 Web、Backend、iOS、Desktop、Cloud、Data、CLI、通用。
11. **风险** - 功能开发的风险点。
12. **状态** - 当前开发状态，如已完成、进行中、未开始。

### 必填行（新功能必写）

以下适配/质量项为新功能必须填写：

| 必填模块 | 适用端/平台 | 说明 |
| --- | --- | --- |
| **多语言/国际化** | 通用 | 新功能必须评估文案、地区格式、时区、货币、RTL 等国际化影响。 |
| **可访问性** | 通用 | 新功能必须评估键盘导航、屏幕阅读器、焦点顺序、语义标签、颜色对比等支持。 |
| **主题/暗黑模式** | 通用 | 新功能必须评估主题 token、颜色、图标、图片资源和对比度。 |
| **埋点/日志** | 通用 | 新功能必须规划业务埋点、工程日志、错误信息和关键耗时。 |
| **监控/告警** | 通用 | 新功能必须规划指标、日志检索、错误率、延迟、资源使用和告警策略。 |
| **安全/隐私** | 通用 | 新功能必须评估鉴权、授权、数据脱敏、敏感日志、合规要求和权限边界。 |

### 使用规范

- 斜体内容为**示例**，编写时替换为实际内容。
- 必填行和必填列不可删除。
- 可根据项目需要新增其余模块行。
- 使用前需复制到自己项目空间后再编写。
